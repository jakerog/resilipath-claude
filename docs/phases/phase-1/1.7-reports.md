## Reporting Architecture on GCP Free Tier

PDF generation cannot run inside a Cloud Run container using headless Chrome (memory-intensive, slow cold start). Instead:

```
Client requests report
  → API enqueues Cloud Task: { exerciseId, format: 'pdf' }
  → Worker Cloud Run service (separate, low-memory) picks up task
  → Worker builds report data from Firestore
  → Worker renders HTML template server-side
  → Worker uses `@react-pdf/renderer` or `pdfkit` (Node-native, no Chrome)
  → Worker uploads PDF to Firebase Storage
  → Worker writes download URL to Firestore: reports/{reportId}
  → Client polls or Firebase listener fires: report ready
  → Client downloads via presigned Firebase Storage URL
```

XLSX export uses `exceljs` (pure Node.js, no Chrome) — can run inline in the API without a worker.
