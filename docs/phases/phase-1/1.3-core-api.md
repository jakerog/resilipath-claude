# Sub-Phase 1.3 — Core API Services
**Version:** 0.2.0  
**Date:** 2026-03-29  

**Sub-Phase Title:** Core API Services  
**Primary Concern:** Implement all REST API endpoints for the 6 core entities and their state machines. This sub-phase contains only API route handlers, service layer business logic, and state machine enforcement. No UI. No real-time. No report generation.  
**Dependencies:** 1.1 (schemas + types), 1.2 (auth middleware)  

---

## API Host

All API routes are served from a **GCP Cloud Run** service running **Hono.js** on Node.js 22.

```
Base URL: https://api.resilipath.app/v1
Runtime: Cloud Run (serverless, free tier: 2M requests/month)
Framework: Hono.js (ultra-lightweight, TypeScript-native, edge-compatible)
```

### 1.3.B — State Machines (Business Logic Only — No I/O)

- [ ] **Task 1.3.B.1:** Implement `TaskStateMachine.canTransition(currentStatus, targetStatus, actor, context): { allowed, reason }`
  - All valid transitions from ADR-006
  - All invalid transitions with specific error codes
  - `TASK_PREDECESSOR_NOT_MET`, `TASK_NOT_ASSIGNED_TO_USER`, `INVALID_STATUS_TRANSITION`

- [ ] **Task 1.3.B.2:** Implement `TaskStateMachine.computeDuration(actualStart, actualEnd, estimatedMinutes)`
  - Returns `{ actualDurationMinutes, varianceDurationMinutes }`
  - Handles null actualEnd (in-progress task)

- [ ] **Task 1.3.B.3:** Implement `StageStateMachine.canActivate(stage, allStageTasks): { allowed, reason }`
  - Stage activates when all non-optional, non-cancelled predecessors complete
  - Rollback stage activation: requires preceding stage `failed` or explicit override

- [ ] **Task 1.3.B.4:** Implement `StageStateMachine.isComplete(stage, allStageTasks): boolean`
  - Returns true when all non-optional, non-cancelled tasks are `completed`

- [ ] **Task 1.3.B.5:** Implement `GoNoGoMachine.processDecision(outcome, task, stage)`
  - `go`: task → completed; returns list of unblocked successor task IDs
  - `no_go`: task → failed; stage → failed; returns rollback stage ID to activate

- [ ] **Task 1.3.B.6:** Implement `ExercisePhaseMachine.canCreateNextPhase(phases): { allowed, nextPhaseName, reason }`
  - Enforces Mock 1 → Mock 2 → [Mock 3 if needed] → Production ordering
  - Production blocked until all required mocks complete

- [ ] **Task 1.3.B.7:** Write unit tests for all state machines — 100% branch coverage
