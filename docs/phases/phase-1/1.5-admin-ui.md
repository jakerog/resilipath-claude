# Sub-Phase 1.5 тАФ Admin UI
**Version:** 0.2.0  
**Date:** 2026-03-29  

**Sub-Phase Title:** Admin UI  
**Primary Concern:** Build the back-end management interface for Admins to create and manage all platform entities (exercises, stages, tasks, resources, teams, vendors, users). This sub-phase covers only management screens тАФ not the live exercise execution interface (that is Sub-Phase 1.6).  
**Dependencies:** 1.1, 1.2, 1.3 (all API endpoints must exist before UI is built)  

---

## Design Constraints

- Framework: **React 18 + TypeScript + Vite** on **Vercel**
- Styling: **Tailwind CSS** + **Radix UI** primitives
- Design language: **Skeuomorphic dark** (as defined in `docs/ux/design-spec.md`)
- Data fetching: **TanStack Query** (all server state via React Query)
- State: **Zustand** (UI-only state: selected exercise, sidebar open, modals)
- No custom components if Radix covers the need
