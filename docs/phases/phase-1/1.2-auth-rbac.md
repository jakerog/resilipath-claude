## Auth Architecture on Firebase (Zero-Cost)

Firebase Authentication is free with unlimited users. ResiliPath uses:
- **Email/Password** — primary login method
- **Google SSO** — one-click login (optional, enabled per tenant)
- **Custom Claims** — role and tenantId injected into the JWT, readable by API and Firestore rules without extra DB lookups

```
Login flow:
  User submits email + password
  → Firebase Auth validates credentials
  → Firebase Auth issues ID token (JWT)
  → Cloud Function (onUserLogin trigger) sets custom claims: { tenantId, role }
  → Client receives token with claims embedded
  → All API requests send token in Authorization: Bearer header
  → API middleware validates token + extracts tenantId + role
  → Firestore Security Rules also validate claims directly
```

### 1.2.B — Custom Claims (Role + TenantId in JWT)

- [ ] **Task 1.2.B.1:** Create Firebase Cloud Function `setUserClaims` (HTTP callable)
  ```typescript
  // Called by Admin when assigning a role to a user
  // Sets custom claims on the user's Firebase Auth token
  exports.setUserClaims = functions.https.onCall(async (data, context) => {
    // Verify caller is admin
    // Set claims: { tenantId: data.tenantId, role: data.role }
    await admin.auth().setCustomUserClaims(data.userId, {
      tenantId: data.tenantId,
      role: data.role
    })
  })
  ```

- [ ] **Task 1.2.B.2:** Create Firebase Cloud Function `onUserCreated` (Auth trigger)
  ```typescript
  // Fires when a new user registers
  // Creates user document in Firestore users/{uid}
  // Assigns default role: 'user' with no tenantId until admin onboards them
  exports.onUserCreated = functions.auth.user().onCreate(async (user) => {
    await db.collection('users').doc(user.uid).set({
      id: user.uid,
      email: user.email,
      role: 'user',
      tenantId: null,  // assigned by admin during onboarding
      status: 'pending',
      createdAt: admin.firestore.FieldValue.serverTimestamp()
    })
  })
  ```

- [ ] **Task 1.2.B.3:** Create Firebase Cloud Function `onUserDeleted` (Auth trigger)
  ```typescript
  // Marks user document as deleted when Firebase Auth user is deleted
  ```

- [ ] **Task 1.2.B.4:** Write Firestore security rule helper function `isRole(role)` that reads custom claims
  ```javascript
  function isRole(role) {
    return request.auth.token.role == role;
  }
  function isTenant(tenantId) {
    return request.auth.token.tenantId == tenantId;
  }
  function isAdminOrModerator() {
    return isRole('admin') || isRole('moderator');
  }
  ```
