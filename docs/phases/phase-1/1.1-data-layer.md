# Sub-Phase 1.1 — Data Layer
**Version:** 0.2.0  
**Date:** 2026-03-29  

**Sub-Phase Title:** Data Layer  
**Primary Concern:** Define and provision the complete Firestore data model — collections, document schemas, indexes, and security rules — that all other sub-phases depend on. No application logic. No UI. Schema only.  
**Dependencies:** None. This is the foundation all other sub-phases build on.  

---

## Firestore Collection Architecture

ResiliPath uses **Firestore Native Mode** with a top-level `tenants/{tenantId}` namespace. All exercise data lives under this namespace, providing strict tenant isolation via Firestore Security Rules.

```
tenants/{tenantId}/
  exercises/{exerciseId}
  exercise_phases/{phaseId}
  exercise_events/{eventId}
  stages/{stageId}
  tasks/{taskId}
  task_predecessors/{predId}
  task_resources/{trId}
  resources/{resourceId}
  teams/{teamId}
  team_resources/{trId}
  vendors/{vendorId}
  evidence_files/{fileId}
  audit_logs/{logId}
  email_lists/{listId}
  email_list_members/{memberId}
  email_templates/{templateId}
  email_schedules/{scheduleId}
  email_deliveries/{deliveryId}

users/{userId}                    ← top-level (Firebase Auth mirrors)
```

### 1.1.C — Firestore Security Rules

- [ ] **Task 1.1.C.1:** Write base tenant isolation rule — all reads/writes require `request.auth.token.tenantId == resource.data.tenantId`

- [ ] **Task 1.1.C.2:** Write `exercises` collection rules
  ```
  read: authenticated user in same tenant
  create: role == 'admin'
  update: role == 'admin'
  delete: role == 'admin'
  ```

- [ ] **Task 1.1.C.3:** Write `tasks` collection rules
  ```
  read: authenticated user in same tenant
  create: role == 'admin'
  update (status/notes/timing fields only): role in ['admin','moderator']
           OR (role == 'user' AND request.auth.uid in resource.data.assignedUserIds)
  update (all fields): role == 'admin'
  delete: role == 'admin'
  ```

- [ ] **Task 1.1.C.4:** Write `stages` collection rules (update rollback activation: moderator+admin only)

- [ ] **Task 1.1.C.5:** Write `resources`, `teams`, `vendors` collection rules (CRUD: admin only)

- [ ] **Task 1.1.C.6:** Write `audit_logs` collection rules
  ```
  read: role == 'admin'
  create: any authenticated user in same tenant (service writes)
  update: DENY ALL
  delete: DENY ALL
  ```

- [ ] **Task 1.1.C.7:** Write `users` collection rules (user can read/update own record; admin can read/update all)

- [ ] **Task 1.1.C.8:** Write `evidence_files` rules (read: same tenant; write: authenticated same tenant; delete: admin)

- [ ] **Task 1.1.C.9:** Deploy security rules to Firebase project via `firebase deploy --only firestore:rules`

- [ ] **Task 1.1.C.10:** Validate rules using Firebase Rules Unit Test suite — write tests for every allow/deny case
